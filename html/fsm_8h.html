◆ fsm_destroy()

void fsm_destroy ( fsm_t *  p_fsm)

Frees the memory allocated in the heap for a given Finite State Machine (FSM).

Nota
Once called this function, the FSM cannot be used again.
You only need to call this function if the FSM was created using fsm_new.
Parámetros
p_fsmpointer to the FSM being destroyed.

◆ fsm_init()

void fsm_init ( fsm_t *  p_fsm,
fsm_trans_t *  p_tt 
)

Configures the initial state of a provided Finite State Machine (FSM) according to its transition table.

Nota
the initial state of the FSM corresponds to the origin state of the first transition of the table
the table must end with a null transition {-1, NULL, -1, NULL}. This is how the library detects the end of the table.
Parámetros
p_fsmpointer to the FSM being initialized.
p_ttppointer to the transition table associated to the FSM.

◆ fsm_new()

fsm_t * fsm_new ( fsm_trans_t *  p_tt)

allocates memory on the heap and returns a pointer to a new Finite State Machine (FSM).

Nota
the initial state of the FSM corresponds to the origin state of the first transition of the table
the table must end with a null transition {-1, NULL, -1, NULL}. This is how the library detects the end of the table.
this function allocates memory in the heap. Once you are done with the FSM, you must call fsm_destroy to free the memory.
Parámetros
p_ttppointer to the transition table associated to the FSM.
Devuelve
fsm_t* pointer to the new FSM.
